# 阶段 5：测试、优化、上线实施方案

## 目标与范围
- 对整个系统（引擎、环境、训练模型、API）进行系统化测试与优化，达成稳定上线标准。

## 测试策略
- 单元测试（Engine）
  - 覆盖 Deck 构建、牌型识别、比较、合法动作生成、计分与特殊规则（必压/顶大/首轮红桃3）。
  - 覆盖率目标：行覆盖 >95%，分支 >85%。
- 集成测试（Env + Agents）
  - RandomAgent/RuleAgent/AI 三者对局流水正确；非法动作在训练期被掩码抑制。
  - 回放序列可复现。
- 接口测试（API）
  - 对局生命周期：start -> 多次 action -> state -> end 全路径。
  - 异常路径：game_id 不存在、非法座位、ActionId 冲突、状态冲突。
- 压力与稳定性
  - 并发会话：≥ 1000 并发对局（按资源调整），平均响应 < 200ms。
  - 长时运行：72h 稳定性验证，无内存泄漏（RSS 曲线平稳）。

## 性能优化清单
- 引擎：
  - 合法动作生成剪枝（按型优先、缓存同构组合）。
  - 排序与比较向量化。
- 推理：
  - 批量推理（batching）与推理队列；超时回退 RuleAgent。
  - 模型蒸馏/裁剪（可选），降低延迟。
- API：
  - 异步 IO；会话对象池；JSON 序列化优化（或 msgpack）。

## 监控与日志
- 指标：QPS、P95/99 响应时间、对局时长、推理耗时、内存/CPU 使用率。
- 日志：请求链路、关键事件、异常栈；对局回放索引（仅保存摘要与种子）。
- 报警：错误率、超时、资源阈值。

## 安全与合规
- 鉴权与限流；防刷策略；输入校验与序列化白名单。
- 模型与权重的访问控制（只读、不可外泄），配置敏感信息通过环境变量注入。

## 上线流程（灰度）
- 环境：预发（staging）-> 小流量灰度 -> 全量。
- 回滚：一键回滚到上一稳定版本；数据库/会话状态无损（API 为无状态，依赖 session store）。
- 版本与变更：SemVer；Changelog；配置与模型版本一一对应。

## 文档与交付
- 使用手册：接口说明、示例请求、错误码。
- 运维手册：部署步骤、监控项、告警处理、回滚流程。
- 训练手册：超参数、数据管线、评测脚本。

## 验收标准（DoD）
- 测试通过率 100%；覆盖率达标。
- 压测与长稳通过；报警与仪表盘上线。
- 文档齐全；上线/回滚剧本演练通过。

## 里程碑
- M1：测试用例补全，覆盖率达标。
- M2：压测与优化达标；监控告警上线。
- M3：灰度与回滚演练通过，正式上线。
