# 阶段 2：AI 训练环境（Gym）实施方案

## 目标与范围
- 将规则引擎封装为标准强化学习环境（Gym 接口），让智能体可与环境交互。
- 提供观测（Observation）、动作（Action）与奖励（Reward）定义，包含动作掩码（Action Mask）。

## 接口设计（不编码，先约定）
- reset(seed?) -> obs
- step(action) -> (obs, reward, done, info)
- observation_space / action_space / action_mask: 通过 info 暴露或包装为 Dict。

## 状态表示（Observation）
- 组成（核心 + 补充）：
  - 我的手牌：45 维二进制向量（是否持有）。
  - 其他两家剩余牌数：2 维数值（归一化到 0-1）。
  - 当前桌面牌（上一手）：牌型类别 One-hot（9）+ 长度 One-hot（最大按规则截断）+ 最大点数 One-hot（13）。
  - 对手历史出牌（最近 3 轮 × 2 名对手）：每轮记录 1 个“类别+长度+最大点数”的组合向量：
    - 类别：9（单/对/三张/三带一/顺子/连对/飞机/带翅膀/炸弹）
    - 长度：8 桶（例如 顺子5-12、连对3-10、飞机2-5；不同类别共享桶位，超出截断）
    - 最大点：13（3..A, 2）
    - 合计每轮 9+8+13=30 维；总计 2×3×30=180 维。
  - 自己已出的牌：45 维二进制或 13×4 计数（推荐 45 位，简洁）。
  - 阶段/轮次：开局/中局/残局 One-hot（3），或 scalar（0-1）；推荐 One-hot 以便表达非线性。
- 总维度示例（参考）：45 + 2 + (9+8+13) + 180 + 45 + 3 ≈ 297 维（不含部分 one-hot 展平差异，最终以实现为准）。
- 归一化：计数/点数类映射到 [0,1]；二进制保持 {0,1}。

## 动作定义（Action）
- **全局动作空间（Global Action Space）**：
  - 动作为 Discrete(N)，N 为所有可能合法出牌的集合大小（估计 2000-5000 左右）。
  - 需要在环境初始化前，预先生成一个**动作映射表 (Action Mapping)**，将每个合法牌型（如“单张3”、“对4”、“顺子34567”）映射为唯一 ID。
  - 动作包含：Pass（ID 0） + 所有单张 + 所有对子 + ... + 所有炸弹。
- **动作掩码（Action Mask）**：
  - 在每一步 `step`，环境根据当前手牌和规则（如必压、首出红桃3），计算当前合法的动作子集。
  - 返回一个长度为 N 的二进制向量（或 indices 列表），供 PPO 策略网络屏蔽非法动作。
- **优势**：相比动态动作空间，固定维度的 Discrete 空间更适合标准 RL 算法收敛。

## 奖励设计（Reward）
- 终局奖励：
  - 胜利：+100
  - 失败：- 剩余牌数（或固定 -100，再加剩余惩罚）
- 中间奖励（可选）：
  - 炸弹成功：+20；被更大炸弹压住则 0。
  - 合法出牌：+0；非法出牌（调试阶段）：-1000 并强终止（训练时不应出现）。
- 惩罚过度行为：
  - 若允许 Pass 的场景，连续无效 Pass 可加入小负奖励（例如 -0.1）。

## Baseline 智能体（用于对比）
- RandomAgent：从合法动作均匀采样。
- RuleAgent：基于启发式规则（能小则小、能管则管、残局保大单/对子等）。
- 评测：每 N 局统计胜率、平均剩余牌数、平均轮数。

## 训练回放与调试
- 保存若干关键对局（obs, action, reward）样本，便于复现与单步排查。
- 可视化：出牌序列与动作概率（用于诊断策略偏差）。

## 风险与缓解
- 动作空间过大：通过剪枝/排序（优先同型合法、再考虑炸弹），并限制历史编码长度。
- 观测泄露：不包含对手手牌等隐藏信息；历史只记录公开信息。

## 里程碑
- M1：环境接口签名冻结；观测/动作/奖励定义确定。
- M2：RandomAgent 与 RuleAgent 评测跑通；回放工具可用。
- M3：与阶段 3 的训练脚手架对接完成。
