# 阶段 6：前端界面开发（Vue 3 + Human-AI 交互）实施方案

## 1. 目标
构建一个基于 Web 的图形化界面，让用户能够直观地与 AI 进行对战。前端将通过 API 与后端游戏引擎交互，展示游戏进程、手牌、出牌动画及最终结算。

## 2. 技术栈
*   **框架**: Vue 3 (Composition API)
*   **构建工具**: Vite
*   **状态管理**: Pinia (管理游戏状态、玩家手牌、当前出牌等)
*   **网络请求**: Axios (调用阶段 4 开发的 FastAPI 接口)
*   **UI 组件库**: Element Plus 或 Naive UI (可选，用于弹窗、按钮等，核心牌桌建议手写 CSS)
*   **样式**: SCSS / Tailwind CSS*   **资源准备**:
    *   **扑克牌图片**: `front/src/porerImg/1.png` ~ `52.png`。
    *   **映射表 (Mapping)**: 需编写一个 JSON 配置文件，将后端的 `Card ID` (Suit/Rank) 映射到前端图片文件名。
        *   后端: 0-44 (共45张)。
        *   前端: 需确认哪张图片对应红桃2、黑桃A等。建议编写脚本自动生成映射。## 4. 功能模块

### 4.1 游戏大厅 (Lobby)
*   **功能**:
    *   输入玩家昵称。
    *   点击“开始游戏”按钮，调用 `POST /game/start`。
    *   显示加载状态（等待 AI 模型初始化）。

### 4.2 游戏主界面 (Game Board)
*   **布局**:
    *   **上方**: 对手 A (AI) - 显示剩余牌数（背面）、头像、昵称。
    *   **左/右方**: 对手 B (AI) - 显示剩余牌数（背面）、头像、昵称。
    *   **下方**: 玩家 (User) - 显示明牌手牌。
    *   **中央**: 出牌区域 - 显示最近一轮打出的牌。
*   **交互**:
    *   **选牌**: 点击手牌上浮/下沉，支持拖拽选牌（进阶）。
    *   **操作按钮**:
        *   **出牌 (Play)**: 校验选中的牌是否符合规则（前端初步校验 + 后端校验），调用 `POST /game/action`。
        *   **过 (Pass)**: 当必须出牌时禁用，允许过牌时可用。
        *   **提示 (Hint)**: (可选) 请求后端计算可行牌型。

### 4.3 结算界面 (Settlement)
*   **触发**: 游戏结束（收到 `game_over` 状态）。
*   **展示**:
    *   胜负结果（胜利/失败）。
    *   各家剩余手牌展示。
    *   **计分板**:
        *   基础分（剩余牌数）。
        *   奖励分（炸弹、春天）。
        *   总变动分。
*   **操作**: “再来一局”按钮。

## 5. 接口对接 (API Integration)
基于阶段 4 定义的 API：
1.  **初始化**: `POST /game/start` -> 获取 `game_id`, `player_id`, 初始手牌。
2.  **轮询/WebSocket**:
    *   定期拉取 `GET /game/state` 更新桌面状态。
    *   或建立 WebSocket 连接接收实时推送（推荐）。
3.  **玩家动作**: `POST /game/action` -> 发送出牌 ID 列表。

## 6. 开发计划
*   [ ] **脚手架搭建**: 初始化 Vue 3 + Vite 项目。
*   [ ] **资源加载**: 编写 `Card` 组件，实现图片 ID 映射与渲染。
*   [ ] **状态管理**: 使用 Pinia 定义 `useGameStore`。
*   [ ] **界面布局**: 实现响应式牌桌布局。
*   [ ] **逻辑对接**: 联调 API，实现发牌、出牌、轮转逻辑。
*   [ ] **动画优化**: 发牌动画、出牌特效、炸弹特效。
*   [ ] **打包部署**: 构建静态文件，集成到后端或独立部署。

## 7. 风险点
*   **图片映射错误**: 需仔细核对后端 `Suit/Rank` 与图片文件的对应关系。
*   **状态同步延迟**: 轮询模式下可能出现卡顿，建议优先考虑 WebSocket。
*   **非法操作处理**: 前端需处理后端返回的“非法出牌”错误，提示用户重新选牌。
