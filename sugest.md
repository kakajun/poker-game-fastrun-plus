问: 训练得分为什么都是负数, 导致训练出来的模型成绩平平

一、先定位训练得分负数的核心原因

从该仓库的代码逻辑（结合扑克AI训练的常见范式）来看，训练得分（回报/奖励值）为负 主要源于「奖励函数设计」「胜负结算逻辑」「强化学习目标」三个核心层面，以下逐一拆解：

1. 奖励函数（Reward）的设计逻辑

扑克游戏的AI训练（尤其是RL/强化学习）中，奖励通常和「对局结果强绑定」：

- 如果对局输了，奖励直接为负（比如 -1/-筹码数）；

- 如果对局赢了，奖励为正（比如 +1/+筹码数）；

- 「平局/弃牌」可能奖励为 0 或小额负数（惩罚无意义操作）。

如果训练过程中模型还处于「学习初期」，决策质量差，输的次数远多于赢的次数，统计层面的「训练得分（平均奖励）」就会呈现负数。

从仓库代码结构来看，核心奖励逻辑大概率在 poker_game/rl/agent.py 或 poker_game/env/poker_env.py 中：

# 示例：扑克环境中典型的奖励结算逻辑（仓库核心逻辑类似）
def step(self, action):
    # 执行动作（下注/弃牌/跟注）
    self._execute_action(action)
    # 判断对局结束
    if self.is_round_over():
        winner = self._get_winner()
        # 给赢家正奖励，输家负奖励
        for player in self.players:
            if player.id == winner:
                player.reward = self.pot / len(winner)  # 赢：正奖励
            else:
                player.reward = -player.bet  # 输：负奖励（亏损筹码）
    return obs, self.reward, self.done, info

2. 强化学习的「累计回报」计算

如果仓库用了 时序差分（TD） / 蒙特卡洛（MC） 等方法，训练中统计的「得分」是「折扣累计回报（Discounted Return）」：

- 公式：$$G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + ... + \gamma^T r_T$$

- 若局内多数步骤的即时奖励 $$r_t$$ 为负，叠加折扣因子（$$\gamma<1$$）后，累计回报 $$G_t$$ 仍会是负数；

- 初期模型探索阶段（ε-greedy策略的ε值高），随机动作导致频繁输局，累计回报必然为负。

3. 「得分」的统计口径问题

仓库中显示的「训练得分」可能不是「胜率」，而是「单局净收益」：

- 比如模型每局平均亏损2个筹码，得分就会显示 -2；

- 即使胜率50%，若「输局的筹码亏损 > 赢局的筹码收益」（比如跟注多、赢局筹码少），得分也会为负。

二、“模型成绩平平”的连锁原因

训练得分（奖励）为负，本质是「模型决策的长期收益为负」，直接导致模型性能差，核心连锁反应：

1. 价值函数（V/Q）拟合偏差

强化学习中，价值函数（比如Q值）是对「未来累计奖励」的估计：

- 若训练数据中奖励长期为负，价值函数会收敛到负数区间；

- 模型会错误认为「所有动作的长期收益都是负的」，无法区分“最优动作”和“差动作”，最终决策随机/低效。

2. 探索-利用（Exploration-Exploitation）失衡

- 初期高探索率（ε）导致大量随机动作，输局多→奖励负→模型不敢探索（ε快速衰减）；

- 模型过早进入“利用阶段”，但此时价值函数未拟合到位，只能重复执行低收益动作，陷入“负奖励→差决策→更负奖励”的恶性循环。

3. 奖励稀疏性+样本分布问题

扑克游戏的奖励是「稀疏的」（只有对局结束才会有奖励），且：

- 若训练集中“输局样本远多于赢局样本”，模型无法学习到“赢局的关键决策”；

- 负奖励的样本占比过高，梯度更新方向偏向“减少亏损”而非“最大化收益”，最终模型只能做到“少输”，而非“赢”，表现为“成绩平平”。

三、针对性优化建议（基于仓库代码结构）

1. 重构奖励函数，增加“中间奖励”

避免仅在对局结束给奖励，给关键决策加小额奖励/惩罚，缓解奖励稀疏：

# 示例：在env/step中增加中间奖励
def step(self, action):
    intermediate_reward = 0
    # 1. 对“弃牌（fold）”的合理时机加奖励
    if action == FOLD and self._is_rational_fold():
        intermediate_reward += 0.1  # 合理弃牌，少亏筹码
    # 2. 对“盲目下注（blind bet）”加惩罚
    elif action == BET and self._is_blind_bet():
        intermediate_reward -= 0.2  # 盲目下注，惩罚
    # 3. 对局结束的最终奖励
    if self.is_round_over():
        final_reward = self._calc_final_reward()
        total_reward = intermediate_reward + final_reward
    else:
        total_reward = intermediate_reward
    return obs, total_reward, self.done, info

2. 调整奖励的“尺度”，平衡正负样本

- 将奖励从「绝对筹码数」改为「相对收益」（比如 (赢的筹码-输的筹码)/总筹码），缩小正负奖励的差距；

- 对赢局奖励做“放大”（比如赢局奖励×1.2），输局奖励做“缩小”（输局奖励×0.8），引导模型向赢局方向学习。

3. 优化训练策略，平衡探索-利用

- 动态调整ε-greedy的ε值：初期缓慢降低ε（比如从1.0到0.1用10万步），让模型充分探索；

- 引入优先经验回放（PER）：让模型更多学习“高收益/高损失”的关键样本，而非平均的负奖励样本。

4. 增加对手多样性，优化训练样本分布

- 不要只让模型和“固定弱对手”训练，加入不同风格的对手（激进/保守）；

- 定期将模型和“历史最优版本”自博弈，增加赢局样本的占比，让价值函数学习到“赢的决策”。

5. 检查评分指标，区分“得分”和“胜率”

- 仓库中若把“单局净收益”作为“训练得分”，负数是正常的（初期输多赢少），需补充「胜率」「筹码累计收益」等指标；

- 比如：即使单局净收益为-0.5，但胜率从10%提升到40%，说明模型在进步，只是还未到“正收益”阶段。

四、总结

训练得分负数的核心是「初期模型决策差→输局多→奖励负」，而“成绩平平”是负奖励导致价值函数拟合偏差、探索利用失衡的必然结果。解决思路的核心是：通过奖励函数重构缓解稀疏性+优化训练策略平衡样本分布+调整指标口径客观评估进度。
