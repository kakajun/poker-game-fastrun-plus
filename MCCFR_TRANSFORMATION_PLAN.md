# MCCFR (蒙特卡洛反事实遗憾最小化) 改造计划

本项目当前使用 **MaskablePPO (强化学习)** 算法，虽然在处理大规模动作空间方面表现良好，但在处理“跑得快”这种**不完全信息博弈 (Incomplete Information Game, IIG)** 时，存在策略难以收敛到纳什均衡、对手隐藏信息建模不足等局限。

本计划旨在引入 **MCCFR (Monte Carlo Counterfactual Regret Minimization)** 算法，使 AI 能够更好地处理博弈中的隐藏信息，并最终实现更强的竞技表现。

## 1. 为什么选择 MCCFR？

| 特性 | 传统 RL (DQN/PPO) | MCCFR (本项目目标) |
| :--- | :--- | :--- |
| **博弈目标** | 最大化累积奖励 | 逼近纳什均衡 (最小化遗憾) |
| **信息处理** | 视对手为环境一部分，难以处理欺骗/诈唬 | 显式建模信息集 (Information Set)，处理对手手牌未知性 |
| **收敛性** | 易陷入局部最优 (只会“少输”) | 在二人/多人博弈中有坚实的数学收敛保证 |
| **鲁棒性** | 对对手策略敏感 | 学习到的策略更难以被对手利用 (Exploitability 低) |

## 2. 改造核心组件

我们将采用 **外部采样 MCCFR (External Sampling MCCFR)**，其核心逻辑如下：

### 2.1 信息集 (Information Set) 建模
- **定义**：玩家在决策时，由于看不见对手手牌，会将多个实际的游戏状态 (State) 视为同一个信息集。
- **改造点**：
    - 在 `src/algo/mccfr/` 下创建 `info_set.py`。
    - 特征提取：包含“我的手牌”、“已出的牌”、“对手剩余牌数”。
    - 使用哈希或桶 (Bucket) 技术对状态空间进行抽象，防止状态爆炸。

### 2.2 遗憾表与策略表 (Regret & Strategy Tables)
- **Regret Table**: 记录每个动作在每个信息集下的累积遗憾值。
- **Strategy Table**: 记录随时间推移累积的平均策略，作为最终的推理模型。
- **存储**：初期使用 `dict` 存储，中后期考虑使用 `Deep MCCFR` (神经网络替代查表)。

### 2.3 递归 CFR 引擎
- 实现 `cfr(h, p_i, p_-i)` 函数，其中 `h` 为游戏历史，`p_i` 为当前玩家概率，`p_-i` 为对手到达该状态的概率。
- 结合蒙特卡洛采样：每次迭代随机采样一套对手的手牌分布。

## 3. 实施路线图 (Phases)

### 第一阶段：原型开发 (Base Implementation)
- [ ] 在 `src/algo/mccfr/` 下实现基础 CFR 逻辑。
- [ ] 适配现有 `src/core/` 引擎，使引擎支持“状态克隆”与“手牌重排”以支持模拟。
- [ ] 编写 `train_mccfr.py` 进行小规模 (如简化牌数) 的收敛测试。

### 第二阶段：状态抽象与优化 (Abstraction & Optimization)
- [ ] **手牌抽象**：将不同的点数组合归类（如 34567 和 45678 视为同类顺子）。
- [ ] **并行化**：利用 Python 多进程加速蒙特卡洛采样过程。
- [ ] **剪枝技术**：引入 Regret Matching+ (RM+) 算法，加速收敛。

### 第三阶段：深度 MCCFR (Deep MCCFR) - 进阶方案
- [ ] 若状态空间过于巨大，将遗憾表替换为深度神经网络。
- [ ] 结合现有 `obs_encoder.py` 的编码能力，作为神经网络输入。

## 5. 预期挑战
- **计算复杂度**：跑得快的动作空间比德州扑克更离散，需要高效的动作生成逻辑。
- **内存占用**：全量遗憾表可能达到 GB 级别，必须进行特征降维。

---
*注：本项目将保留 PPO 训练管线作为基准 (Baseline)，通过 MCCFR 训练出的策略将与其进行对弈评估。*
